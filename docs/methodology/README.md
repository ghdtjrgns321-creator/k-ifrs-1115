# Methodology — AI 개발 의사결정 기록

> 이 폴더는 프로젝트의 모든 주요 의사결정을 **"문제인식(Painpoint) → AI 활용 해결 → 기술 선택 논리"** 3단 서사로 복원·기록한다.
> 목적: 면접에서 어떤 구간을 질문받아도 "왜 이렇게 만들었는지"를 본인 언어로 설명할 수 있는 상태.
> 코드 한 줄 단위의 정합성 검수가 아니라, **결정 단위**의 논리 검증이 대상이다.

## 작성 규칙

1. 유닛 1개 = 파일 1개. `_TEMPLATE.md`의 6개 섹션을 모두 채운다.
2. 정량 수치는 반드시 출처(테스트 스크립트 경로, 이력서, PROJECT_OVERVIEW.md)를 병기한다. 출처 없는 수치는 쓰지 않는다.
3. "다시 한다면" 섹션은 비워두지 않는다 — 한계를 말할 수 있어야 무비판 수용이 아니다.
4. 작성 주체는 사용자 본인. AI는 초안 검증과 면접관 역할(꼬리질문)만 담당한다.
