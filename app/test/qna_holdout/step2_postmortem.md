# STEP 2 실패 분석 — 혼동쌍 기반 프레임 판별은 왜 안 되는가

작성 2026-07-07. 결론: **혼동쌍을 기계적으로 분리하는 신호가 존재하지 않아** STEP 2(혼동 구조화)를 현재 형태로는 채택하지 않는다. 관련 데이터(`confusion_pairs.json`)와 전처리(`19-ontology-confusion.py`)는 기록·재검토용으로 보존, 런타임 주입은 무효화했다.

## 1. 무엇을 고치려 했나 — D버킷(프레임 오선택)

미재현 14건 중 D버킷 4건: 정답 문단이 컨텍스트에 회수됐는데 LLM이 **다른 개념 프레임**으로 답한 실패.

대표 케이스 **QNA-SSI-35567(반품조건부 판매)**:
- 정답 프레임 = 변동대가/거래가격 산정(문단 47·53·56)
- LLM 답변 = 재매입약정 프레임(부록B 문단 B65~72) — 완전 오선택
- "반품조건부"와 "재매입"이 둘 다 "재화가 되돌아온다"는 시나리오라 실무자도 혼동하는 전형.

설계 가설: 혼동 가능한 개념쌍(반품권↔재매입)을 온톨로지에서 기계 도출해 두고, 그 쌍이 동시에 관련될 때 "둘 중 하나를 확정하고 나머지는 배제하라"는 **[프레임 판별]** 헤더를 컨텍스트 상단에 주입하면 LLM이 옳은 프레임을 고를 것.

## 2. 시도한 3가지 활성화 신호 — 전부 실패

혼동쌍 사전(413쌍: 형제 191 + e3 222)은 만들 수 있었다. 문제는 **"이 질문에서 어느 쌍이 진짜 혼동인가"를 판별하는 신호**였다. 세 가지를 실측했고 모두 무효였다.

### 시도 A — 구조(형제 개념쌍): 노이즈 폭발

"같은 부모의 형제 개념이 동시 진입하면 혼동쌍"으로 정의. 실측(캡처 14건, 진입 concept_ids 기준):

| 케이스        | 진입 개념 수 | 형제쌍 발동 수 |
| ------------- | ------------ | -------------- |
| QNA-SSI-35567 | 20           | **15**         |
| QNA-SSI-38676 | 25           | **19**         |
| QNA-SSI-38680 | 15           | 14             |

**실패 이유**: 진입셋이 subtree 확장으로 형제를 도배한다. 케이스당 15~19쌍이 발동하면 "모든 게 혼동"이라 헤더가 무의미하다. 부록B 적용지침 한 부모에만 형제쌍이 91개.

### 시도 B — 의미(개념 임베딩 코사인): 신호가 반대

"구조상 형제이면서 의미적으로 유사한 쌍만 진짜 혼동"으로 좁히려 함. STEP 1에서 만든 개념 임베딩(80개, 4096-dim)으로 반품권과 모든 혼동 파트너의 코사인 측정:

| 파트너                       | 반품권과 코사인      |
| ---------------------------- | -------------------- |
| 환불부채                     | 0.791                |
| 고객이 행사하지 않은 권리    | 0.741                |
| 보증                         | 0.684                |
| 위탁약정                     | 0.656                |
| **재매입약정 (정답 혼동쌍)** | **0.559 ← 최하위권** |

**실패 이유**: 정답 혼동쌍(재매입)이 오히려 가장 낮게 랭크됐다. 기준서 본문상 반품권(환불부채·추정 제약)과 재매입(재매입가격·금융요소)은 **텍스트가 다르다**. 실무자의 혼동은 "재화가 물리적으로 되돌아온다"는 **시나리오 차원**인데, 이는 조문 텍스트에도 개념 임베딩에도 반영되지 않는다. 코사인은 이 혼동을 잡을 수 없다.

### 시도 C — via_topic 게이팅: 비결정적으로 놓침

"LLM이 직접 지목한 주제(via_topic)에 두 개념이 다 들어야 혼동"으로 정의 → 노이즈는 0~2쌍으로 수렴(성공). 35567 원본 캡처에서 via_topic=[반품권, 재매입, 거래가격] → 헤더 정상 발동.

그러나 **35567을 3회 재실행**하니:

| 재실행 | via_topic에 반품권 | via_topic에 재매입 | 헤더 발동  | 답변이 인용한 재매입 문단 |
| ------ | ------------------ | ------------------ | ---------- | ------------------------- |
| 1      | O                  | **X**              | **미발동** | B65, B70~72               |
| 2      | O                  | **X**              | **미발동** | B70~72                    |
| 3      | O                  | **X**              | **미발동** | B65, B66, B70, B72        |

**실패 이유**: analyze(LLM)가 비결정적이라 재매입을 via_topic에 **때마다 다르게** 뽑는다. 3회 모두 재매입이 topic에서 빠졌고, 그럼에도 답변은 재매입 문단을 인용했다. 정밀한 신호였지만 신호가 나타나야 할 때 사라진다.

## 3. 더 깊은 근본 원인 — 오염은 e3 hop에서 온다

결정적 확인: 재실행 1에서 재매입(vw8tzF)은 via_topic도 concept_ids(진입셋)도 **아니었다**. 그런데 재매입 문단 B65가 답변에 인용됐다.

경로 추적 결과 B65는 **e3 문단 상호참조 hop**으로 컨텍스트에 딸려왔다 — 재매입 개념이 진입한 게 아니라, 진입한 다른 개념의 문단이 B65를 e3로 가리켜 hop-이웃으로 끌려온 것.

→ **D버킷 오염은 개념 진입이 아니라 문단 참조 그래프에서 샌다.** 진입·혼동쌍 어느 레이어로도 못 막는 이유가 여기 있다. "넓게 다 뿌리고 LLM이 고르게" 아키텍처(문단 상한 없음 + e3 hop 확장)의 구조적 대가다.

## 4. 결론과 남은 레버

혼동쌍 식별은 세 신호(구조·의미·via_topic) 모두 무효 + 오염이 e3 hop에서 새므로, **현재 형태로는 채택하지 않는다.** 손으로 혼동쌍을 적는 것은 오버피팅이라 배제.

남은 두 레버(둘 다 진입에 신호를 더하는 방식이 아님):
1. **회수 폭 축소** — e3 hop 확장을 끄거나 hop-이웃 문단 프루닝으로 오염을 근원 차단. 리스크: 재현율 하락(넓은 진입이 baseline 96.5%의 원천).
2. **생성 프레임 규율** — 컨텍스트를 개념별 섹션으로 그룹핑해 LLM이 "이 문단들은 재매입, 질문은 재매입 아님"을 구조로 보게. 혼동쌍 식별 불필요, 오버피팅 없음, 항상 적용. 효과는 dev 측정 필요.

D버킷의 현실적 상한은 1~2건(35567 + 아마 38675)이며, 나머지 미재현의 무게중심은 기타 6건(소프트축)이라는 점도 함께 기록한다.

## 4-B. 레버2(개념별 섹션 그룹핑) 실험 — 기각 + 35567 재해석 (2026-07-07 추가)

레버2를 실제 구현·측정했다. 컨텍스트를 관할 개념 title별 섹션(`### 개념: 재매입약정` 등)으로
묶어 프레임 경계를 LLM에 구조로 노출. 개념 title 그룹핑은 프레임을 깨끗이 갈랐다(변동대가·거래가격=gold
vs 재매입약정·풋옵션=오답, 실측 확인).

**35567 3회 재실행 결과: 3/3 여전히 풋옵션/재매입 프레임 인용(B65·B70~72), gold(47·53·56) 교집합 0/3.**
개선 없음 → 사전 공약(Δ>0)에 따라 기각. 코드 롤백(generate.py 무변경 상태 복원).

산출물: `exp_grouping_r{1,2,3}.json`.

### 왜 그룹핑이 안 먹혔나 — 35567은 "프레임 오선택"이 아니었다

측정 중 근본 재해석이 나왔다. 35567 질문 = **"판매 후 2년 내 반품 신청 시 최초가격 80%로 되사는 정책"**.
이건 기준서상 **반품권과 재매입약정(풋옵션)이 진짜로 교차 연결된** 케이스다:

- 판단트리(match_judgment_tree)가 이미 정답 규칙을 주입 중(1409자): **"【풋옵션】 재매입가가 원래
  판매가격 미만이면: 유인이 유의적이면 리스, 유의적하지 않으면 반품권 판매(B20~B27)처럼 처리(B72)"**.
- 질문은 80%(판매가 미만) → LLM은 이 규칙을 정확히 읽고 재실행 답변에서 **Case2=반품권**에 도달한다.
- 즉 LLM은 엉뚱한 프레임을 잡은 게 아니라, 기준서가 명시적으로 이어놓은 풋옵션→반품권 경로를 걷는다.
  is_situation=True(clarify)로 라우팅돼 "경제적 유인이 유의적인가?"를 되묻는 조건부 답변을 낸다.
- "미재현" 판정의 실체 = 인용 문단이 풋옵션 규칙 문단(B70~72)이라 gold(47/53/56)와 겹치지 않을 뿐.
  회계적으로 LLM 답변은 오히려 정밀하다(gold는 반품권 default를 전제, LLM은 그 default의 조건을 명시).

**결론: 35567은 회수·진입·그룹핑 어느 레버로도 못 고친다. 고칠 대상이 아니다** —
기준서 자체의 개념 교차연결 + 인용중첩 채점의 산물이다. D버킷 대표 케이스가 이렇다는 것은
D버킷 미재현의 상당수가 "결함"이 아니라 채점 아티팩트·기준서 모호성일 수 있음을 시사한다.

### 최종 D버킷 판정

진입 3신호(구조·의미·via_topic) 기각 + 출처 라벨링(hop/주인미진입) 실측 무효(35567 캡처에서 재매입이
via_topic으로 진입, B65가 직할 문단이 됨) + 생성 레이어 그룹핑 3/3 무효 + 대표 케이스 재해석 결과
"고칠 대상 아님". **전 레이어에서 기각. D버킷 종료.** 미재현의 무게중심은 기타 6건(소프트축)으로 확정.
