# 컨텍스트: 감리사례 기반 위험신호(Red Flag) 체크리스트 작성

## 이 문서의 목적

금융감독원 감리지적사례를 분석하여,  
꼬리질문 시스템에 사용할 **"위험신호 경고형 체크리스트"**를 함께 만드는 것입니다.

**작업 결과물**: `app/domain/red_flags.py` 파일의 `RED_FLAG_PATTERNS` dict 항목

---

## 배경: 감리사례 체크리스트의 독특한 목적

| 구분 | 본문 체크리스트 | QNA 체크리스트 | **감리사례 체크리스트** |
|------|--------------|--------------|----------------------|
| **역할** | 판단 요소 확인 | 전제조건 매칭 | **위험 패턴 경고** |
| **톤** | 중립적 ("~인가요?") | 중립적 | **경고형** ("~한 상황이라면 위험합니다") |
| **목적** | 올바른 처리 안내 | QNA 적용 가능성 확인 | **과거 지적 패턴과 현 상황 대조** |

감리사례 꼬리질문은 단순히 사실관계를 확인하는 것이 아니라,  
**"당신의 상황이 과거에 지적받은 패턴과 유사하지 않은지"** 를 경고하는 목적입니다.

---

## 감리지적사례의 구조 이해

현재 시스템에서 감리사례 데이터는 아래 형식으로 저장되어 있습니다:

```
# 배경 및 질의
[회사가 적용한 회계처리 방식]

# 감리지적내용  
[금감원이 지적한 올바른 회계처리와 지적 이유]

관련 기준서: K-IFRS 1115호 문단 XX
```

이 데이터에서 **"어떤 상황에서 어떤 잘못된 처리를 했는가"** 패턴을 추출합니다.

---

## 작업 방법

### Step 1: 지적 패턴 유형 분류
감리사례 데이터에서 반복적으로 등장하는 위반 패턴을 아래 유형으로 분류해주세요:

| 패턴 ID | 위반 유형 | 설명 |
|--------|---------|------|
| `매출_조기인식` | 수익 인식 시점 오류 | 통제 이전 전에 매출 인식 |
| `수행의무_하나로_묶기` | 수행의무 과소 분리 | 구별되는 용역을 단일 의무로 처리 |
| `변동대가_과대계상` | 변동대가 제약 무시 | 반품·리베이트 가능성 미반영 |
| `본인_대리인_역전` | 순액/총액 오분류 | 대리인인데 총액 인식 (또는 반대) |
| `라이선스_시점_오류` | 라이선스 인식 시점 | 기간 인식해야 하는데 시점 인식 |
| `계약변경_미처리` | 계약변경 미반영 | 가격/범위 변경 시 수익 미조정 |

### Step 2: 위험 패턴 → 진단 질문 변환

각 패턴에서 **"이 지적을 받은 회사가 처했던 상황적 특징"** 을 추출하세요.  
그 특징을 꼬리질문으로 변환하면, 유사한 위험이 있는지 미리 경고할 수 있습니다.

```
패턴명: [위반 유형]
지적 이유: [금감원이 지적한 핵심 이유]
위험 신호 (상황 특징): [이 지적을 받은 회사들의 공통 상황]
진단 질문: [유사 위험이 있는지 확인하는 꼬리질문]
근거 조항: [K-IFRS 1115호 문단 번호]
```

### Step 3: Python dict 형식으로 출력

```python
RED_FLAG_PATTERNS = {
    "패턴명": {
        # 이 패턴을 탐지할 키워드 (사용자 질문 또는 검색된 감리사례에서)
        "trigger_keywords": ["키워드1", "키워드2"],
        
        # 이 패턴의 위반 요약
        "violation_summary": "어떤 잘못된 회계처리인지 한 줄 요약",
        
        # 관련 K-IFRS 1115호 조항
        "related_paragraphs": ["35", "38"],
        
        # 위험 신호 진단 질문
        # 각 질문은 "이 상황이라면 위험하다"는 패턴 확인용
        "red_flag_questions": [
            {
                "question": "사용자에게 물어볼 진단 질문",
                "risk_if_yes": "Yes라면 어떤 위험이 있는지",
                "paragraph_basis": "근거 조항"
            },
        ],
        
        # 꼬리질문 앞에 붙는 경고 메시지
        "warning_prefix": "⚠️ 유사 감리 지적 사례가 있습니다. 아래를 확인해주세요."
    }
}
```

## 꼬리질문 톤 가이드라인

감리사례 꼬리질문은 본문/QNA 꼬리질문과 톤이 달라야 합니다:

| 구분 | 톤 | 예시 |
|------|------|------|
| 본문 체크리스트 | 중립, 탐색적 | "재고위험은 누가 부담하나요?" |
| QNA 체크리스트 | 중립, 확인적 | "반품 시 현금 환불인가요?" |
| **감리사례** | **경고적, 예방적** | "⚠️ 출고일과 매출인식일이 다른 경우가 있나요?" |

사용자를 위협하거나 단정 짓지 않되,  
**위험 신호를 인지시키는 것** 이 목적입니다.

---

## 주의사항

- 감리사례의 세부 회사명, 금액은 체크리스트에 포함하지 마세요 (익명성 유지).
- 지적 패턴의 **핵심 구조**만 추출하여 일반화된 질문으로 만드세요.
- 기준서 근거가 없는 지적 패턴은 `# 근거 조항 확인 필요` 주석으로 표시하세요.
- `risk_if_yes`는 단정이 아닌 "가능성" 표현으로 작성하세요. ("~위험" / "~의심 패턴")
