# 컨텍스트: 본문 & 적용지침 기반 판단 체크리스트 작성

## 이 문서의 목적

K-IFRS 제1115호 **본문 조항과 적용지침(부록 B)**을 근거로,  
꼬리질문 시스템에 사용할 **"판단 트리형 체크리스트"**를 함께 만드는 것입니다.

**작업 결과물**: `app/domain/decision_trees.py` 파일의 `DECISION_TREES` dict 항목

---

## 배경: 이 체크리스트가 어디에 쓰이는가

**K-IFRS 1115호 챗봇**의 꼬리질문 생성 시스템에 사용됩니다.

흐름:
1. 사용자가 애매한 상황을 자연어로 질문
2. RAG가 관련 기준서 조항을 검색하여 AI가 1차 답변 생성
3. AI가 답변과 함께 **꼬리질문**을 제시하여 사용자의 상황을 구체화
4. 꼬리질문을 반복하며 사용자의 불완전한 질문 → 완성된 판단 가능한 상황으로 업그레이드

**꼬리질문의 목적**: 회계 처리 판단에 필요한 사실관계 중 아직 확인되지 않은 항목을 질문

**환각 방지 원칙**: 체크리스트 항목은 반드시 K-IFRS 1115호 조항에 명시된 판단 기준만 사용.  
기준서에 근거 없는 질문은 추가 금지.

---

## 작업 방법

### Step 1: 주제 목록 정의
아래 주제들이 현재 우선순위입니다. 각 주제별로 체크리스트를 만들어주세요.

| 주제 | 핵심 조항 | 우선순위 |
|------|----------|---------|
| 본인 vs 대리인 | B34~B38 | 🔴 최우선 |
| 변동대가 (반품권 포함) | 50~55, B20~B27 | 🔴 최우선 |
| 수행의무 식별 (구별성) | 22~30, B2~B13 | 🔴 최우선 |
| 기간에 걸쳐 vs 한 시점 인식 | 35~38 | 🔴 최우선 |
| 라이선스 | B52~B63 | 🟡 중요 |
| 고객충성제도 (포인트/마일리지) | B39~B48 | 🟡 중요 |
| 보증 (확신형 vs 용역형) | B28~B33 | 🟡 중요 |
| 본인부담금/할인권 (비현금대가) | 66~69, 70~72 | 🟢 추가 |

### Step 2: 각 주제별 정보 수집
각 주제를 분석할 때 아래 항목을 채워주세요:

```
주제명: [예: 본인_vs_대리인]
트리거 키워드: [이 주제를 탐지하는 키워드들]
핵심 조항: [해당 문단 번호들]
판단 결론: [Yes/No로 갈리는 최종 판단이 무엇인가]
판단에 필요한 사실관계: [각 판단 기준별 확인 필요 항목]
기준서 근거: [각 항목이 몇 조에 근거하는지]
```

### Step 3: Python dict 형식으로 출력
아래 스키마를 반드시 준수하세요:

```python
DECISION_TREES = {
    "주제명": {
        # 이 주제를 탐지할 키워드 (사용자 질문 또는 검색된 문서에서 탐지)
        "trigger_keywords": ["키워드1", "키워드2"],
        
        # 관련 K-IFRS 1115호 문단 번호 (참조 근거)
        "related_paragraphs": ["B34", "B35", "B36"],
        
        # 꼬리질문으로 변환될 체크리스트
        # 각 항목 = 사용자에게 물어볼 사실관계 질문
        # 뒤에 "(문단 XX)" 형식으로 기준서 근거 반드시 표시
        "checklist": [
            "질문 내용 (문단 B34)",
            "질문 내용 (문단 B35~B36)",
        ],
        
        # 이 체크리스트의 판단 목표 (대화 마지막에 결론 지을 사항)
        "judgment_goal": "판단하려는 최종 회계 처리 결론"
    },
}
```

## 체크리스트 항목 작성 기준

1. **질문 형식**: 사용자에게 직접 물어보는 형태 ("~인가요?", "~은 누구인가요?")
2. **Yes/No 또는 단답 응답 가능**: 복잡한 서술 요구 금지, 사실관계 확인에 집중
3. **K-IFRS 공식 용어 사용**: "통제", "수행의무", "변동대가", "이전" 등
4. **문단 번호 반드시 표기**: 각 항목 끝에 `(문단 B34)` 형식으로 근거 조항 명시
5. **판단 영향도 순서**: 판단에 가장 영향이 큰 항목을 리스트 앞에 배치

## 출력 예시 (참고용, 실제 조항 검토 후 수정 필요)

```python
"본인_vs_대리인": {
    "trigger_keywords": ["본인", "대리인", "위탁판매", "수수료", "중개"],
    "related_paragraphs": ["B34", "B35", "B36", "B37", "B38"],
    "checklist": [
        "다른 당사자가 재화/용역을 제공하기 전에 해당 재화/용역을 통제합니까? (문단 B35)",
        "재화나 용역의 가격을 설정할 재량권을 갖고 있습니까? (문단 B37 나목)",
        "재화/용역이 이전되기 전과 반품 시 재고 위험을 부담합니까? (문단 B37 라목)",
        "이행에 대한 주된 책임(이행책임)은 누가 집니까? (문단 B37 가목)",
    ],
    "judgment_goal": "총액 인식(본인) vs 순액 인식(대리인) 결정"
},
```

## 주의사항

- 이 파일을 읽은 LLM은 조항을 **직접 읽고** 체크리스트를 작성해야 합니다.
- 기억 속의 지식으로 작성하지 말고, 가능하면 조항 원문을 확인 후 작성해주세요.
- 불확실한 항목은 `# TODO: 조항 확인 필요` 주석으로 표시하세요.
- 완성된 항목도 코드에 `# 근거: 문단 B35` 인라인 주석을 달아주세요.
